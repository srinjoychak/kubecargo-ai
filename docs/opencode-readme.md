# K8s Automation_Ansible - K8s Offline Deployment with OpenCode AI

Automated offline package downloader for air-gapped Kubernetes deployments using K8s Automation_Ansible.

---

## Overview

This toolset uses OpenCode AI to automatically download all required files (binaries, container images, RPM packages) for deploying Kubernetes on air-gapped RHEL/AlmaLinux servers. The AI resolves all component versions, researches live RPMs, and packages them into the correct archives.

### Centralized Skill Architecture
To ensure consistency across all AI models (Claude, Gemini, Codex, OpenCode), the download logic is centralized in:
`skills/k8s_downloader_core.md`

---

## End-to-End Deployment Flow

```
1. Pull K8s Automation_Ansible code from GitHub
            |
2. Verify local environment (Linux + Internet + Docker/Skopeo)
            |
3. Run AI Skill to download K8s packages (Generates binaries, images, packages)
            |
4. Update extra_vars-gen.json with cluster details
            |
5. Run ansible_execute.sh to deploy the cluster
```

## Step-by-Step Guide

### Step 1 - Get K8s Automation_Ansible Code
Pull the K8s Automation_Ansible automation code onto your deployment/staging server.

```bash
git clone https://github.com/srinjoychak/kubecargo-ai.git
```

### Step 2 - Verify Execution Environment
The AI skill requires a **Linux host with internet access**. If your current machine is Windows or macOS:
- SSH into a Linux build server or VM.
- OR run the AI within a Linux container (optional).

### Step 3 - Download K8s Packages via AI Skill
Open your AI assistant (Claude, Gemini, or OpenCode) and use this prompt:

```
Download all packages for Kubernetes v1.34.6 offline deployment targeting el9 on root@10.127.199.185 at /root/K8s_Automation_Ansible
```

**What the AI does:**
- Resolves CoreDNS, etcd, and Pause versions from the K8s source tree.
- Researches current RPM versions from AlmaLinux mirrors to prevent 404s.
- Runs `scripts/k8s_offline_downloader.sh` with the correct arguments.
- Produces `binaries.tar.gz`, `images.tar.gz`, `packages.tar.gz`, and `download_manifest.json`.

| Archive | Contents | Typical Size |

| `binaries.tar.gz` | kubeadm, kubelet, kubectl, etcd, helm, containerd, crictl, cfssl, socat, keepalived | ~115 MB |
| `images.tar.gz` | Core K8s + Calico images (stored as Raw Tar for containerd import) | ~400 MB |
| `packages.tar.gz` | containerd RPM, chrony RPM, keepalived + perl dependency RPMs, shared libraries | ~65 MB |
| `other/` | cfssl, cfssljson, yq, calico manifest, tar/unzip/zip RPMs | varies |
